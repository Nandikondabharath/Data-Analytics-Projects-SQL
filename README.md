# Food Delivery Data Analytics - SQL Project

## Project Overview

This is a comprehensive **Food Delivery Platform Analysis** project that demonstrates advanced SQL queries and data analytics capabilities. The project analyzes a food delivery service ecosystem comprising customers, restaurants, and orders across multiple Indian cities.

**Author:** N. Bharath Reddy  
**Data Range:** January 2025 - May 2025 (May is partial - 1 day only)  
**Dataset Size:** 200 orders | 50 customers | 10 restaurants

---

## 📁 File Descriptions

### 1. **N_Bharath_Reddy_SQL_Project.sql**
The core SQL project file containing:
- **Database & Table Creation:** Defines the `food_delivery` database schema with three main tables (customers, restaurants, orders)
- **15 Advanced SQL Queries:** Demonstrates a range of SQL techniques including:
  - JOINs (INNER, LEFT JOIN)
  - Aggregation functions (SUM, COUNT, AVG)
  - Window functions (RANK, LAG, ROW_NUMBER)
  - CTEs (Common Table Expressions)
  - CASE statements for categorization
  - Subqueries and HAVING clauses
- **Key Insights:** Detailed analysis of platform metrics and actionable recommendations

### 2. **customers.csv**
Customer master data with 50 records containing:
- `customer_id` - Unique customer identifier (1-50)
- `customer_name` - Full name of the customer
- `city` - City where customer is located (Hyderabad, Chennai, Bangalore, Delhi, Mumbai)
- `signup_date` - Date customer registered on the platform (ranging from Jan 2024 - Dec 2024)

### 3. **orders.csv**
Transaction data with 200 order records containing:
- `order_id` - Unique order identifier (1-200)
- `customer_id` - Foreign key linking to customers table
- `restaurant_id` - Foreign key linking to restaurants table
- `order_date` - Date order was placed (Jan-May 2025)
- `order_amount` - Order value in currency (ranging from ₹151 to ₹1,196)
- `payment_method` - Payment type used (Debit Card, Credit Card, UPI, Cash)
- `order_status` - Completion status (Completed or Cancelled)

### 4. **restaurants.csv**
Restaurant master data with 10 records containing:
- `restaurant_id` - Unique restaurant identifier (1-10)
- `restaurant_name` - Name of the restaurant
- `city` - City location (Delhi, Bangalore, Mumbai, Hyderabad)
- `rating` - Customer rating on a scale of 1-5 (ranging from 3.7 to 5.0)

## 🔍 SQL Queries & Analysis

The project includes 15 progressive SQL queries:

### Query 1: Completed Orders with Details
Identifies all completed orders along with customer and restaurant names with order amounts.

### Query 2: Total Revenue by Restaurant
Calculates total revenue generated by each restaurant to identify top performers.

### Query 3: Customer Order Count
Counts the total number of orders placed by each customer for engagement metrics.

### Query 4: Average Order Value by City
Computes average order amounts for each restaurant city to understand spending patterns.

### Query 5: Order Value Categorization
Classifies orders into Low (< ₹200), Medium (₹200-₹500), and High (> ₹500) tiers.

### Query 6: Restaurants Above Platform Average
Identifies restaurants that outperform the platform-wide average order value.

### Query 7: Customers with No Orders
Finds customers who signed up but never placed an order (churn risk).

### Query 8: Top 5 Customers by Lifetime Spending
Ranks customers by total spending to identify high-value users.

### Query 9: Most Popular Payment Method
Determines which payment method is most frequently used.

### Query 10: Daily Revenue with Running Total
Shows daily revenue trends and calculates cumulative revenue over time using window functions.

### Query 11: Restaurant Rankings by Revenue
Ranks restaurants by total revenue generated using the RANK() window function.

### Query 12: Second-Highest Order Amount
Finds the second-highest order value placed on the platform.

### Query 13: Above-Average Customers
Identifies customers who place more orders than the average customer.

### Query 14: Top 3 Restaurants by Order Volume per City
Uses CTEs and window functions to find top-performing restaurants in each city.

### Query 15: Month-over-Month Revenue Trend
Compares monthly revenue with previous months using LAG() function to track growth trends.

## 📊 Key Insights

### 1. **HIGH CANCELLATION RATE - CRITICAL ALERT** ⚠️
- **94 of 200 orders (47%)** are Cancelled; only 106 (53%) Completed
- This is the single biggest red flag - nearly 1 in 2 orders fail to complete
- **Recommendation:** Investigate cancellation clusters by restaurant, payment method, or time window to identify root causes

### 2. **REVENUE CONCENTRATION IN ONE RESTAURANT**
- **Tandoori Nights** generates ₹26,602 (19.75% of total platform revenue)
- Total platform revenue: ₹134,756
- Single restaurant drives nearly 1/5 of all revenue
- **Recommendation:** Study Tandoori Nights' success drivers (pricing, rating, delivery speed) to replicate with mid-tier restaurants

### 3. **70% OF RESTAURANTS UNDERPERFORM ON ORDER VALUE**
- Platform-wide average order value: **₹673.78**
- Only 3 restaurants exceed average:
  - Tandoori Nights: ₹782.41
  - BBQ Nation: ₹744.44
  - Spice Hub: ₹735.92
- **Recommendation:** Provide menu pricing optimization and upsell support to underperforming restaurants

### 4. **SIGNIFICANT CITY-LEVEL ORDER VALUE GAP**
- Mumbai (highest): ₹735.92 average order value
- Hyderabad (lowest): ₹610.67 average order value
- **Gap: 20.5%** between top and bottom city
- **Recommendation:** Investigate whether gap reflects restaurant mix, customer income profile, or delivery radius differences

### 5. **100% CUSTOMER ACTIVATION**
- All 50 signed-up customers have placed at least one order
- Zero dormant signups in the dataset
- **Insight:** Retention isn't the problem - order completion is the key focus area

### 6. **TOP 5 CUSTOMERS DRIVE 19% OF REVENUE**
- Top 5 customers (10% of customer base) contributed ₹25,847 (19.18% of revenue)
- Led by Nandini Rao with ₹5,894 lifetime spending
- **Recommendation:** Implement loyalty/retention programs targeting this VIP segment

### 7. **DEBIT CARD IS DOMINANT PAYMENT METHOD**
- Debit Card: 63 orders (31.5%)
- UPI: 24% | Cash: 23% | Credit Card: 21.5%
- Fairly balanced payment mix overall
- **Recommendation:** Ensure Debit Card payment infrastructure is highly reliable since it handles the largest share

### 8. **HEALTHY REPEAT USAGE PATTERNS**
- Average orders per customer: 4
- 19 of 50 customers (38%) exceed average
- Maximum: 8 orders (Nandini Rao)
- **Insight:** Meaningful share of user base shows strong repeat engagement, not just a handful of outliers

### 9. **REVENUE PEAKED IN MARCH, THEN DECLINED**
- January: ₹30,933
- February: ₹32,772 (+6%)
- March: ₹38,650 (Peak, +18% vs Jan)
- April: ₹31,111 (-19.5% from peak)
- May: ₹1,290 (Note: Only 1 day of data - exclude from trend analysis)
- **Insight:** April decline warrants investigation; May data incomplete for conclusions

## 📈 Database Schema

### Customers Table
| Column | Type | Description |
|--------|------|-------------|
| customer_id | INT | Primary key |
| customer_name | VARCHAR(100) | Customer name |
| city | VARCHAR(100) | City location |
| signup_date | DATE | Registration date |

### Restaurants Table
| Column | Type | Description |
|--------|------|-------------|
| restaurant_id | INT | Primary key |
| restaurant_name | VARCHAR(100) | Restaurant name |
| city | VARCHAR(100) | City location |
| rating | FLOAT | Customer rating (1-5) |

### Orders Table
| Column | Type | Description |
|--------|------|-------------|
| order_id | INT | Primary key |
| customer_id | INT | Foreign key to Customers |
| restaurant_id | INT | Foreign key to Restaurants |
| order_date | DATE | Order placement date |
| order_amount | DECIMAL(10,2) | Order value |
| payment_method | VARCHAR(50) | Payment type (Debit Card, Credit Card, UPI, Cash) |
| order_status | VARCHAR(50) | Status (Completed or Cancelled) |
